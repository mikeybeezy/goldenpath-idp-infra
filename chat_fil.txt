basd on the below. provide feedback 

If Kong is your ingress/API gateway (recommended default)

Use an NLB in front of Kong.

Why:

NLB is simple, fast, and stable for exposing a single entrypoint (Kong)

Works well for TCP + HTTP(S), gRPC, WebSockets

You let Kong own L7 routing/policies (auth, rate limits, routes), not AWS

Typical pattern: Service type=LoadBalancer for Kong → NLB → Kong routes to services.

If you want AWS-native L7 ingress (without Kong doing the L7 heavy lifting)

Use an ALB.

Why:

Best for HTTP/HTTPS host/path routing at the AWS edge

Easy integration with AWS WAF, redirects, listener rules, OIDC auth (AWS-native)

Clean “Ingress resource → ALB” flow

Quick decision rule

Kong as the single front door → NLB

AWS-managed L7 routing/WAF as the front door → ALB

Don’t run both by default in V1 unless you have a specific reason (it creates ambiguity/support load)
